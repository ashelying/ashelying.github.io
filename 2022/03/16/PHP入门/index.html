PHP变量

PHP 变量规则：

  • 变量以 $ 符号开始，后面跟着变量的名称
  • 变量名必须以字母或者下划线字符开始
  • 变量名只能包含字母、数字以及下划线（A-z、0-9 和 _ ）变量名不能包含空格
  • 变量名是区分大小写的（$y 和 $Y 是两个不同的变量）,建议用小写
  • 变量首次为其赋值时自动创建
  • 不必声明数据类型

PHP文件
  • 打开文件
    • r 只读。在文件的开头开始。
    • r+ 读/写。在文件的开头开始。
    • w 只写。打开并清空文件的内容；如果文件不存在，则创建新文件
    • w+ 读/写。打开并清空文件的内容；如果文件不存在，则创建新文件
    • a 追加。打开并向文件末尾进行写操作，如果文件不存在，则创建新文件。
    • a+ 读/追加。通过向文件末尾写内容，来保持文件内容。
    • x 只写。创建新文件。如果文件已存在，则返回 FALSE 和一个错误。
    • x+ 读/写。创建新文件。如果文件已存在，则返回 FALSE 和一个错误
  • 文件的基本操作
    • copy(源文件，目的文件) 复制文件
    • unlink(目标文件) 删除文件
    • rename(旧文件名，新文件名) 重命名文件或目录，或移动文件
    • ftruncate(目标文件资源，截取长度) 将文件截取到指定长度
    • file_exists(目标文件名) 判断文件或文件夹是否存在
    • is_file(文件名) 判断指定的路径存在且为文件
  • 目录的基本操作
    • mkdir(pathname) 新建一个指定的目录
    • rmdir(dirname) 删除目录，该目录必须为空
    • getcwd(void) 取得当前文件所在的目录
    • chdir(dirname) 改变当前目录
    • opendir(path) 打开目录，返回目录的指针
    • closedir() 关闭目录，参数为目录指针
    • readdir() 遍历目录
    • scandir(path,sort) 以数组形式遍历目录，sort可设升序或降序

Cookie和Session

作用：记录用户状态的技术（http本身无状态）

  • cookie

    • 设置cookie
      • Cookie在传输过程中，时HTTP协议头的一部分，用户浏览器和服务器之间传递信息
      • 任何属于HTML文件本身的内容输出之前调用**SetCookie()**函数
        • SetCookie(name,value,expire,path,domain,secure)
        • 必填：name(cookie名字)，value(cookie值)
        • 可选：expire(有效期)，path(服务器端的有效路径),domain(设定有效域名),secure(设定是否仅在https安全连接时才设定)
    • 注意事项
      • SetCookie()之前不能有任何html输出
      • SetCookie()后，在当前页调用echo $_COOKIE[“name”]不会有输出，需刷新或在下一个界面
      • 限制，一个浏览器能创建的Cookie最多为30个，每个不超过4KB，每个web站点设置COOKIE总数不超过20个
      • COOKIE存在客户端，用户禁用即失效
      • cookie生命周期到即失效
      • 设置一个cookie失效时间为time()在页面浏览完之后失效

  • Session

    • 概述
      • 从进入网页到浏览器关闭，即用户浏览网站所花费的事件
      • 可用户身份认证，程序状态程序，页面之间的参数传递
      • 保持客户端的状态

  • Cookie和Session异同

    ​

    异同点 Cookie Session
    相同 保存用户信息
    不同 存储位置 存放在客户端浏览器 存放在服务器
    保留时间 设定有效保存期限 关闭浏览器自动作废
    存储容量 每个浏览器保存某个web最多20个不超过4KB 可创建多个，会影响服务器性能
    是否被禁用 用户设置是否开启 不会被禁用，内容保存在服务端
    隐私策略 对客户端可见不安全 储存在服务器，较安全
    跨域支持 支持跨域名访问 不支持
